О проекте "Traceroute as a Service"

Это учебный веб-проект по дисциплине "Компьютерные сети". Он представляет собой полноценный сервис, который позволяет пользователям выполнять системную команду traceroute прямо из браузера и получать ее результаты в удобном формате.

Цель проекта — продемонстрировать понимание того, как работают IP, ICMP, DNS-резолвинг, как измеряются задержки и строится маршрутизация пакетов через промежуточные узлы. Бэкенд написан на Python с использованием фреймворка FastAPI. Для хранения истории запросов и маршрутов применяется база данных SQLite, к которой происходят асинхронные запросы через SQLAlchemy и aiosqlite.

Помимо парсинга стандартного вывода команды traceroute, сервис обращается к базе данных GeoLite2, чтобы для каждого публичного IP-узла определять страну и город. Данные отображаются не только в виде таблицы, но и визуализируются на интерактивной карте с подключением библиотеки Leaflet.js.

Проект обернут в Docker для возможности развертывания на любом Linux сервере, а запросы к FastAPI перенаправляются через reverse-proxy Nginx.

Как запустить проект

Для запуска понадобится установленный Docker и плагин Docker Compose.

Сначала необходимо клонировать этот репозиторий на вашу локальную машину или удаленный сервер. После клонирования перейдите в корневую папку проекта.

Опционально (для отображения стран на карте) потребуется скачать бесплатную базу данных GeoLite2. Нужно создать в корне проекта директорию data и положить в нее файл GeoLite2-City.mmdb. Если файл не скачать, проект все равно запустится и будет работать, но без привязки к геолокации.

Когда структура и директория готовы, выполните команду для сборки образов и поднятия контейнеров в фоновом режиме:

docker compose up -d --build

Docker скачает необходимые образы, установит Python библиотеки из requirements.txt прямо внутрь контейнера, поставит утилиту traceroute и запустит uvicorn и nginx.

После успешного запуска проект будет доступен по локальному адресу http://localhost/ (или по порту 80 вашего внешнего сервера). Если потребуется внести изменения в код, после их сохранения будет необходимо перезапустить контейнер, чтобы изменения применились в докере.
